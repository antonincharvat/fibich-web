From Empty Window to First Playback

Today Fibich crossed an important line: it can play music.

Today Fibich crossed an important line: it can play music.

That sounds like the most obvious thing a music player should do, but getting there changes the whole feeling of the project. Until now, Fibich has mostly been a shape: a native macOS window, the beginning of a player interface, and the first ideas around how a local music library should be represented. Now there is sound coming out of it.

Recently, I started building the first real structure around the app. Albums, artists, tracks, and library items are now becoming the basic vocabulary Fibich uses to understand a music collection. That matters because the goal is not to create a tiny utility that opens one audio file and forgets about it. The goal is a proper macOS music player for local libraries, lossless albums, carefully stored folders, and collections that deserve a bit more respect.

The library setup is also starting to take shape. Fibich can now be pointed toward a music location, generate a simple library, and let tracks become part of the player experience. It is still early, but this is the beginning of something important: the app is starting to connect files on disk with a music-focused interface.

Today's bigger step is playback itself.

Fibich now has basic audio playback, track progress, seeking, and volume control. These are small words for features that make a large difference. A progress bar tells you the app knows where it is in the track. Seeking means you can move through the music instead of just waiting. Volume control makes the player feel immediate, like something you can actually use rather than just look at.

This is still a rough first version, but it is the first version that feels alive.

There is a lot ahead. Playlists need to become more durable. Metadata will need proper attention. The library will need to handle real collections, not only clean test folders. macOS integration will matter too: media keys, now playing information, Finder workflows, and all the small details that make an app feel native rather than merely present on the desktop.

But today, the center is there.

Fibich is beginning to become what it is meant to be: a native hi-fi audio player for macOS, focused on local music files, wide format support, library organization, and a listening experience that respects the user's own collection.

The first sound is not the destination. But it is the point where the project starts to feel real.
